# RelaxMap QA Testing

Manual QA testing project for the RelaxMap web application.

## About

This repository contains QA documentation created during the testing of the RelaxMap web application.

Testing was performed based on project requirements and Figma designs, with a focus on functionality, UI/UX, user flows, and responsive behavior across different devices.

## Testing Scope

- Functional testing
- UI/UX testing
- Responsive testing
- Cross-device testing
- Positive and negative testing
- Requirements validation

## Test Environment

- macOS
- Google Chrome
- Chrome DevTools
- Desktop — 1440 × 900
- iPad Mini — 768 px
- iPhone SE — 375 px

## QA Documentation

### Checklists

Test checklists cover selected application areas and user states:

- Layout (Header and Footer) — Unauthorized User
- Layout (Header and Footer) — Authorized User
- Registration Page — Unauthorized User
- Home Page — Unauthorized User
- Home Page — Authorized User
- Add Review Modal — Authorized User

The checklists include functional, UI, navigation, and responsive testing across mobile, tablet, and desktop devices.

### Bug Reports

Detailed bug reports include:

- Bug ID
- Environment
- Severity and priority
- Preconditions
- Steps to reproduce
- Actual result
- Expected result
- Attachments
